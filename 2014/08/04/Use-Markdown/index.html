使用Markdown书写感慨

八月 4 2014 | 评论

Reading makes a full man, conference a ready man, and writing an exact man. -Bacon 读书使人充实，讨论使人机敏，写作使人精确。 ——培根

我个人是比较讨厌写东西的，因为写得也不好。不过既然搞了博客，就要keep bloging。近期由于工作关系，需要交接手头的所有工作。因为时间紧迫，没法一一和对接人面对面说明我的所有工作内容，于是打算趁着周末将交接内容逐条写下来给他。就是这个事情，让我深刻体会到了培根大叔“写作使人精确”的深刻内涵了。

之前是写在为知笔记里，使用类似todolist的样子然后写给他，哪些做了就勾掉，还有哪些没做，但是发现todolist方式写出来思路比较零散，写着写着容易忘记在写啥玩意，也没空去找一个更好的TodoList相关软件写给他。想想使用思维导图应该是个很好的主意，但是担心对接人可能不习惯所以没有用。由于之前写博客的关系，于是我打算用Markdown语法直接写下来生成html给他。

谁知道这一用起来简直让我停不下来。我使用的是Cmd Mardown编辑阅览器来编辑，可以很直观地看到我写的markdown生成的页面和大纲，让我写作思路清晰、连贯，让我认真回顾了一下自己的工作内容和经历。

也许你会说Word也行嘛。确实，用Word书写确实也可以生成大纲，但是实际操作来说，Word总有着这个或那个的缺点（读者可自行搜索，笔者是随便搜的两篇）。从我的角度出发，我感觉是Word需要手动排版才能显示出大纲，而且Word的排版对于写作这个过程来说，实际上是打断了写作思路，特别是像我这种很容易被别的事情吸引过去的人，手一移到鼠标上，后面就不知道要干啥了。

你可能会说完全可以不排版嘛，你用txt直接书写也是一样的呀。从我个人角度出发，看到有格式的相关文字，感觉条理性会更好，不知道读者有没有这样的共鸣。而且你也可以直接在txt上用Markdown语法写作嘛。增加一点点符号和规则就能让你写出来的内容有着美妙的大纲和排版，何乐而不为呢。

还有人会说，你可以用大名鼎鼎的LaTeX嘛。这玩意真是专门写学术东西的神器，可是——太难啦……学起来耗时，写起来耗力……放弃之。

总的来说，从这个写交接内容的经历来看，使用Markdown写东西，确实感到这样思路就会很清晰，而且可以脱离鼠标，写起来思路更连贯顺畅。而且使用类似Word书写总是要考虑怎么排版好看，格式刷都要用到烂了，但是Markdown基本不用什么考虑排版问题……这也就是为什么很多博客平台我都放弃了，它们居然不支持Markdown语法！（PS.可能还有一个原因是Markdown里有Mark吧，lol）

说了那么多，就给大家分享一些Markdown语法和工具的介绍把。

Markdown教程：

  • Markdown快速入门
  • Markdown 语法说明 以上两篇教程都是出自http://wowubuntu.com ，作者写得十分清晰简单，如果想快点开始写作，只看第一篇就行了。

Markdown编辑器：

  • Cmd Markdown编辑阅览器
  • Sublime Text + MarkdownEditing 强烈推荐第一个，可能网上还会推荐别的比如MarkdownPad之类的软件，但是说实话，都不如Cmd Markdown。所见即所得、快速、有教程案例、支持自定义……像我这些只用写文字内容的，或者偶尔来电代码的人来说，足够了。

后记

用了Markdown写作之后，让我稍微坚定了Keep bloging的想法。后续看来要学习双拼，已提升自己的打字速度，以便更好滴keep bloging。
