又用Linux

11/11/2014 | 评论

折腾之神再次附身，我再次对着自己的电脑折腾起来——这次折腾的有是自己电脑的操作系统。

初遇Linux

大约08年的时候初次接触到Linux这个东西，当时正是Ubuntu推出其里程碑般的桌面版本Ubuntu 8.04LTS展现出其另一番风貌的时候。大学时候赶时髦也用虚拟机搞了一个Ubuntu 8.04LTS来耍耍。不过那时候Linux毕竟不是很好用，当时耍也仅仅是好奇，想看看另一套著名的操作系统桌面版究竟是什么样子。大学期间，还把自己的电脑折腾成3系统，其中一个操作系统就是Ubuntu。不过由于当时用起来很多当时需求无法满足，而且使用起来几乎每步操作我都得谷歌一下，费时费力，最终这个Linux启动项沦落成为我的电脑装X的一个存在罢了。

几乎没咋使用的使用经历

在初识Linux到工作期间，虽然不论是工作还是生活上几乎和Linux没有什么大的关系，但是身为伪程序员，不用Linux怎么好意思。当时初衷是向从桌面版出发，如果平时工作都能代替Windows，那么距离服务器版也不远了嘛。于是在那段时间一直在尝试使用各种排名靠前的Linux（多亏了虚拟机），找寻自己喜欢的Linux。最终的结果就是我终于发现自己被微软强X得是多么的彻底，我几乎所有事情都无法脱离Windows，否则就寸步难行。

这段试用期间，发现了LinuxMint，它是一款可以理解为是Ubuntu的美化版。LinuxMint用起来很顺手，但是毕竟不是国产，有很多不是很舒适的地方。当时用得感觉比较好的就是YLMF OS（现在叫做startos），而后发现的Deepin也和它不分伯仲。但是遗憾的是由于种种原因，最终我的本本还是完全被Windows霸占，连启动项都没了，仅剩下几十G的空间给以后的Linux。没想到的是这一留就是一年多。

再次尝试Linux

可能是因为自己本本安装了太多补丁，本本的Win7开机极慢，运行一些软件也有变慢趋势，虽然更换了一台更好的本本，但是问题并未解决。于是又蠢蠢欲动地开始去四处找Linux来充实自己的电脑。当时更想尝试既可以用作桌面版本又可以用来作为服务器的版本，于是自然放弃了Ubuntu等专攻桌面版本的Linux系列，转而投向REDHAT之类的版本。在美观、稳定性方面对比了很久，最终本本中第二个操作系统使用了OpenSUSE，虚拟机中使用CentOS以备对比。

OpenSUSE确实可以说是稳定性和美观并重。但是如果要用来代替Windows，看来任重而道远，还是有很多操作需要不断地百度一下。最主要还是软件方面问题，比如QQ、迅雷等等常用软件在OpenSUSE下使用起来不顺手，于是OpenSUSE被我冷落在旧本本中。虚拟机中的CentOS使用起来感觉更不适合作为Windows的替代品，很多软件都不如意，最后CentOS也被冷落在虚拟机中。

最后还是考虑，新手的话还是老老实实从最简单的Ubuntu系列开始耍吧。遂找来deepin、Ubuntu、Linux Mint等尝试安装，结果由于本本问题，不论是安装哪个版本安装时引导它都出现(initramfs) Unable to find a medium containing a live file system。网上查了一下说是请检查CDROM是否插到SATA3接口上。这个答案太不靠谱了吧，我的电脑是本本，总不可能为了安装你个操作系统要拔下光驱吧？搞了几天还是没法安装，于是这个想法暂时搁置了。

遇到LinuxMint

近期OpenSUSE、CentOS纷纷发布新版本，看到新闻的我再次激起心中的折腾欲。但是之前一致担心(initramfs)错误，不过想想还是再研究研究。首先尝试了感觉可能是最符合国情的deepin，欣然发现deepin中的U盘制作工具解决了我本本无法安装的问题，于是很高兴地安装试用起来。结果用了没两天，感觉很不习惯。Deepin默认采用的桌面系统并不是我喜欢的KDE，操作习惯与WINDOWS差距太大，而且默认软件等也感觉不爽，像我这种被windows强X过久的人一时半会儿没能缓过来，还是放弃了Deepin……

由于目标是Ubuntu的嫡系版本，又一时不知道ylmf os改名叫做StartOS了，干脆就直接换上了著名的Linux Mint。不愧是http://distrowatch.com/网站上排名第一的linux，或者说世界排名第一的桌面Linux操作系统——Linux Mint，使用起来简便快捷，看来如果不出意外，Linux Mint暂时可以作为我的Windows替代品。

　写在后面

本来想在本篇还写上一些教程、说明或者实际具体操作流程。不过看到前面太多感慨内容，所以本篇还是作为一篇经历感文算了，下一篇再做教程。
