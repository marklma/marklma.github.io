Groovy脚本的编码问题

八月 22 2013 | 评论

编码问题本以为当我全都是用utf-8或者全部使用gbk的话应该就不会出现了，结果在学习groovy中还是让我碰到了。

我使用notepad++运行以下代码的时候报错了，代码内容是：

··· groovy File INPUT_FILE = new File('E:/Documents and Settings/mark/桌面/') println(“hi中文”) println(INPUT_FILE) println(INPUT_FILE.isDirectory()) INPUT_FILE.eachFileMatch(~/huawei.*.cfg/) { println it.name
} ···

运行结果：

hi中文
E:\Documents and Settings\mark\桌面\

false
Caught: java.io.FileNotFoundException: E:\Documents and Settings\mark\桌面java.io.FileNotFoundException: E:\Documents and Settings\mark\桌面 at mark.main.BatchHw.run(BatchHw.groovy:16)

这样的结果让我很吃惊，居然说我输入的不是目录。我把文件名换成“.”发现没问题，后来测试发现只有文件路径中包含中文就会出现问题，此时我才意识到是可恶的编码问题。

我将notepad++里面编码全部统一为了UTF-8，nppexec里面输入输出编码我也弄成了统一UTF-8，但是还是失败，经过多次切换ANSI、gbk之后发现还是不成功。实在没办法就求助IDE——eclipse的帮助。使用eclipse创建project并编译后执行发现没有任何问题，即使是执行groovy script也成功地显示结果，但是用notepad++运行却仍然失败。

求助搜索引擎大神后终于找到了结论。 groovy -h中有这么一条 -c,--encoding specify the encoding of the files 嗯于是将nppexec运行脚本改为了cmd /c groovy -c utf-8 “$(FULL_CURRENT_PATH)“，运行，done！

结论：

groovy编译时候不会根据.groovy文件的编码自行判定，而是需要人为指定编码才可按照指定编码运行。网上大多倾向于这是groovy的一个小bug。
