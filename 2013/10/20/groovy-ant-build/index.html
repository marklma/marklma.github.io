Groovy与ANT配置

十月 20 2013 | 评论

捎带手学习了一下如何使用ANT，尝试了gradle，发现功能貌似过于强大，而且编译啥的好像可以集成ANT来使用，于是决定先学习使用常见的构建工具ant。本文中使用的环境是： Groovy Version: 2.1.7 JVM: 1.7.0_40 Vendor: Oracle Corporation OS: Windows 7 Apache Ant™ version 1.9.2 log4j-1.2.17：放在\anttest\lib目录下，工程目录为\anttest\

ant配置文件： \anttest\build.xml

<?xml version="1.0" encoding="utf-8"?>
<project name="HelloWorld" basedir="." default="main">

 <property environment="env"/>
 <property name="src.dir"  value="src"/>
 <property name="lib.dir"  value="lib"/>
 <property name="build.dir"   value="build"/>
 <property name="classes.dir" value="bin"/>
 <property name="jar.dir"  value="${build.dir}/jar"/>

 <property name="main-class"  value="mark.main.HelloWorld"/>
 <property name="groovyScript"  value="mark/main/HelloWorld"/>

 <path id="classpath">
  <fileset dir="${lib.dir}" includes="***.jar"/>
 </path>

 <mkdir dir="${classes.dir}"/>

 <taskdef name="groovy"
   classname="org.codehaus.groovy.ant.Groovy"
   classpathref="groovy.classpath"/>
 <taskdef name="groovyc"
   classname="org.codehaus.groovy.ant.Groovyc"
   classpathref="groovy.classpath"/>

 <target name="clean">
  <delete dir="${classes.dir}"/>
  <delete dir="${build.dir}"/>
 </target>

 <target name="compileJava" description="compile java to bytecode">
  <mkdir dir="${classes.dir}"/>
  <copy todir="${classes.dir}">
   <fileset dir="${lib.dir}"/>
  </copy>
  <javac srcdir="${src.dir}" destdir="${classes.dir}" includeantruntime="on" classpathref="groovy.classpath"  encoding="utf-8">
  </javac>
  <copy todir="${classes.dir}/">
   <fileset dir="${src.dir}">
    <include name="**"/>
    <exclude name="***.groovy"/>
   </fileset>
  </copy>
 </target>

 <target name="compile" description="compile groovy to bytecode" depends="compileJava">
  <groovyc srcdir="${src.dir}" destdir="${classes.dir}" includeantruntime="on" classpathref="groovy.classpath" encoding="utf-8">
   <classpath location="${classes.dir}"/>
  </groovyc>
 </target>

 <target name="jar" depends="clean,compile">
        <mkdir dir="${jar.dir}"/>
  <copy todir="${jar.dir}">
   <fileset dir="${env.GROOVY_HOME}/embeddable" includes="groovy-all-*.jar" excludes="*-indy.jar"/>
  </copy>
        <jar destfile="${jar.dir}/${ant.project.name}.jar" basedir="${classes.dir}">
            <manifest>
                <attribute name="Main-Class" value="${main-class}"/>
            </manifest>
        </jar>
    </target>

 <target name="runGroovyScript" depends="compileJava">
  <groovy src="${src.dir}/${groovyScript}.groovy" classpathref="groovy.classpath">
   <classpath location="${classes.dir}"/>
   <arg value="${action}"/>
  </groovy>
 </target>

 <target name="run" depends="compile">
  <java classname="${main-class}" classpathref="groovy.classpath">
   <classpath location="${classes.dir}"/>
   <arg value="${action}"/>
  </java>
 </target>

 <target name="main" depends="clean,run"/>
</project>

其中在task runGroovyScript中，我在org.codehaus.groovy.ant.Groovy中并未找到encoding字样的内容，但是我在groovyc中找到了encoding，果然是无处不在的乱码问题，是不是官方的一个小纰漏呢？有什么方法可以执行groovy -c utf-8这样字段的方法？请各位赐教。

测试.groovy脚本： \anttest\src\mark\main\HelloWorld.groovy

package mark.main

import org.apache.log4j.Logger;
import org.apache.log4j.BasicConfigurator;

println("HelloWorld");
Logger logger = Logger.getLogger(HelloWorld.class);
BasicConfigurator.configure();
logger.info("Hello World");
