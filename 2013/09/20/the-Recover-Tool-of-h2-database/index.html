H2 Database的Recover Tool

九月 20 2013 | 评论

前段时间做的一个grails页面在服务器重启后突然挂了，TOMCAT跑不起来，各种报错，正好赶上近期Grails升级到了Grails-2.3.0版本，于是先尝试使用Grails-2.3.0，结果升级后试用结果各种碰壁，为了解决Grails2.3.0的问题，将Groovy升级到了2.1.7，将JDK改用了1.7，将所有路径改成了没有空格的，修改各种PATH、CLASSPATH，结果无功而返，不是缺少这就是缺少那，最后实在没辙了还是改用回了Grails-2.2.4。结果居然还是报错！但是新建项目没问题，而是旧项目出现了问题，纠结了半天才想到，是不是那个著名的“内存数据库”挂了。

经过测试果不其然，确实是数据库挂了。想想数据库总有恢复功能吧，网上找了一下，没啥中文说明，最后还是在官方页面找到了解决方法。在此记录下方法：

用记事本打开对应的trace文件，发现错误建议为请我使用recover tool解决，结果网上找了半天没见到哪有recover tool下载或者使用说明。官网也没有下载，看了看官方FAQ，原来是自带功能。将损坏的数据库devDb.h2.db移动到h2的bin目录下，如\h2\bin，然后进入cmd后进入对应目录输入： java -cp h2-1.3.172.jar org.h2.tools.Recover

于是可以看到bin目录下生成了一个SQL文件devDb.h2.sql。运行h2.bat之后，设置好数据库进入后将devDb.h2.sql中的内容写入数据数据库，大功告成……

由此看出h2 database虽然方便易用，但是确实稳定性不佳，仅仅是服务器重启后就无法恢复，还需要人为手动处理后才恢复。早知道之前不偷懒使用mysql就好了……
