Grails学习笔记（1）:Grails如何删除领域，如何使用grails Generate-all

九月 3 2013 | 评论 

grails create-app racetrack
grails create-domain-class Race
grails create-controller-class Race

使用grails常见生成代码如上，生成是能生成，但是怎么删除呢？使用grails help查看发现也没有delete或者remove代码。

网上查了一下，没有自动删除的代码，只能手动删除，拿grails create-domain-class Race这一句为例，仅仅需要删除racetrack/test/unit/racetrack/RaceTests.groovy和racetrack/grails-app/domain/racetrack/Race.groovy

注；Race为类，racetrack为项目名和默认包名 来源：http://stackoverflow.com/questions/2221610/removing-controllers-and-domain-classes-in-grails

错误代码： ··· grails generate-all User grails generate-controller User grails generate-views User ···

生成了领域类User之后，输入以上代码输入可能会出现 错误：No domain class found for name User. Please try again and enter a valid domain class 实际是因为我没有输入默认的包名。grails 2.1如果没输入报名创建类的话，会默认以项目名为默认包名，所有类均默认在该包下，错误已经指出使用domain class，本例中的项目名仍为racetrack，所以正确应该是： ··· grails generate-all racetrack.User grails generate-controller racetrack.User grails generate-views racetrack.User ···
